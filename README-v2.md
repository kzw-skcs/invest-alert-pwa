# invest-alert-pwa v2 — 完全版アップグレード (2026-07-02)

## v2で何が変わったか

| 機能 | v1 | v2 |
|---|---|---|
| 買い判定 | チャネル±3%帯の6状態 | 6状態は継続 + **Value多因子スコア0-100**(チャネル位置1年/5年・高値からの下落率・RSI・マクロ/個別判別・長期トレンド健全性) → 👀監視50 / 🟦購入検討65 / 🔥強い買い80 / 🚨絶対的買い場90 |
| 暴落の質 | 判定なし | **マクロ暴落(=Value好機)と個別悪材料(=falling knife)を自動判別**(対S&P500相関×VIX×相対下落幅)。個別要因は-15点減点 |
| モメンタム | なし | trade銘柄: **50日高値ブレイク+20/50MA+RSI 50-72+対S&P500相対強度+出来高1.5倍**でエントリーシグナル |
| 損切り | なし | trade銘柄: **初期-8%上限・ATR×2.5**の自動ストップ、ATRトレーリング、50日線割れ/急落6%/デッドクロスの売り管理、8週停滞タイムストップ |
| 部分利確 | なし | trade: **+20%で1/3、+40%でさらに1/3**、残りトレーリング。hold: チャネル上限+RSI75過熱+比率超過で✂️トリム提案 |
| hold銘柄の売り | なし(仕様) | 損切りは今も出さない。代わりに**🧐テーゼ崩壊見直し**(5年チャネル下抜け+200日線下向き20日継続) |
| ポートフォリオ | 株6:BTC1:ETH1:金1:銀1 | **株60:BTC10:ETH5:金10:銀5:戦略キャッシュ10** + 株内サブセクター目標(AI半導体45/電力DC15/エネルギー15/宇宙防衛15/モート10) |
| 配分 | なし | **💰配分タブ**: 運用可能総額→フル配分プラン(段階的エントリー40/30/30付き)、追加投資額→ギャップ×シグナルで投入先を金額提示 |
| IPO | なし | **🚀IPOタブ**: Anthropic/OpenAI IPOのフェーズ管理・恩恵銘柄アクション・ロックアップ監視。config.jsonのeventsで管理 |
| 銘柄 | 30銘柄 | +ORCL(Stargate)、+CRWV(OpenAIコンピュート)。全銘柄に確信度(conviction)とサブセクター付与 |
| 再分析 | 的中率+帯調整 | **Value/モメンタム別の的中率**、minScoreToAlert・帯幅・stopInitPctを安全範囲内で自動調整 |

## 更新手順(従来通り)
```bash
cd ~/invest-alert-pwa && git checkout -- . && git pull origin main
SRC=$(ls -dt ~/Downloads/invest-alert-pwa-v2*/ | head -1)
cp -R "$SRC". ~/invest-alert-pwa/
cd ~/invest-alert-pwa && git add -A && git commit -m "v2アップグレード" && git push
```
- 配布ZIPに **data.json / analysis.json / signals_log.json は含まれません**(実データを壊さないため)
- push後、GitHub → Actions → **daily → Run workflow** を1回手動実行するとv2形式のdata.jsonが生成されます
- Vercelは自動デプロイ。PWAのService Workerはv2キャッシュへ自動切替(旧キャッシュ全削除)

## 保有データについて(重要)
- v2はlocalStorageキーを `iv2.*` に変更。初回起動時に旧データを自動スキャンし、見つかれば取込ダイアログを表示します
- 見つからない場合は再入力が必要です。以後は💼資産タブの**バックアップ(エクスポート/インポート)**をご利用ください

## 各アラートの発生場所
- **サーバー(毎朝6時・push対象)**: 絶対的買い場/強い買い/購入検討、モメンタムエントリー、売り管理、テーゼ見直し、トリム、IPOイベント、VIX
- **クライアント(アプリ表示時)**: 取得単価ベースの損切り/第1・第2利確/トレーリング割れ、リバランス提案
  ※取得単価は端末内のみ保存のため、単価依存アラートはアプリを開いた時に計算されます

## 任意設定(v1から未設定のまま)
1. プッシュ通知: `npx web-push generate-vapid-keys` → 公開鍵を `vapid.json` に `{"publicKey":"..."}` で配置、GitHub Secretsに `VAPID_PRIVATE_KEY` / `VAPID_CLAIMS_EMAIL`
2. アプリからのサーバー同期・再分析: Vercel環境変数 `GH_TOKEN`(repo+workflow権限PAT) / `GH_REPO=kzw-skcs/invest-alert-pwa`
